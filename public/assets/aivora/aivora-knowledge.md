# Aivora — Product Knowledge Base

## What is Aivora?

Aivora is a super-intelligent autonomous multimodal AI OS built entirely by Fokrul Islam. It runs as a Next.js web application with no dependency on OpenAI, Anthropic, Ollama, Groq, or any external AI API. All AI inference happens either in the browser (via WebGPU-accelerated WebLLM) or through an optional user-supplied OpenAI-compatible endpoint.

Aivora is designed for privacy-first, offline-capable, document-grounded question answering. It combines a hybrid vector + keyword retrieval pipeline with a multi-step reasoning agent loop to deliver cited, grounded answers from a user's own Knowledge Vault.

## Core Architecture

Aivora's server is a Next.js App Router application. The AI pipeline consists of five layers:

1. **Embedding Layer** — Uses `@xenova/transformers` (Xenova/all-MiniLM-L6-v2, 384 dimensions) to generate dense vector embeddings locally on the server. No embedding API calls are ever made to external services.

2. **Vector Store** — Supabase PostgreSQL with the `pgvector` extension. Chunks are stored in a `document_chunks` table with 384-dimensional vectors. A companion `documents` table holds document metadata.

3. **Hybrid Retrieval** — A Supabase RPC function `match_document_chunks_hybrid` performs weighted hybrid search: 70% vector similarity + 30% pg_trgm keyword match. Default match count: 8. Similarity threshold: 0.35.

4. **Reasoning Agent Loop** — A five-stage loop: Plan → Retrieve → Reflect → Self-Correct → Respond. The loop is orchestrated in `src/lib/ai/agents/aivora-agent.ts`. Stages use lightweight prompts that optionally call a server LLM or fall back to built-in logic.

5. **Answer Generation** — Server LLM (optional, OpenAI-compatible endpoint) or browser-local Phi-3.5-mini (WebLLM). When neither is configured, Aivora still retrieves chunks and returns them to the browser for WebLLM generation.

## Modes of Operation

Aivora operates in three modes:

- **Local WebLLM Mode** (default when Supabase is configured, AI_CHAT_* not set): Full hybrid RAG retrieval runs on the server. Answer generation runs in the browser using Phi-3.5-mini via WebGPU. No external LLM API needed.

- **Full RAG Mode** (Supabase + AI_CHAT_BASE_URL + AI_CHAT_API_KEY configured): Server handles both retrieval and generation. Supports any OpenAI-compatible endpoint (e.g., LM Studio, self-hosted vLLM, Mistral API).

- **Aivora OS Lite** (Supabase not configured): Vector retrieval is unavailable. Browser WebLLM handles general questions. The UI labels this mode "Aivora OS Lite" and prompts the user to connect Supabase.

## Reasoning Agent Loop — Detailed

**Plan:** The planner (`src/lib/ai/reasoning/plan.ts`) decomposes the user query into search intents and a retrieval plan. It uses the server LLM if available, otherwise applies rule-based decomposition.

**Retrieve:** The hybrid retriever fetches matching chunks from Supabase using pgvector cosine similarity and pg_trgm keyword scoring. Results are merged with weighted scoring.

**Reflect:** The reflection module (`src/lib/ai/reasoning/reflect.ts`) evaluates retrieval quality: confidence score, whether context is weak, whether the query is out of scope for available documents.

**Self-Correct:** If reflection signals weak retrieval, the self-corrector (`src/lib/ai/reasoning/self-correct.ts`) rewrites the query and triggers one retry retrieval pass.

**Respond:** Top-5 chunks (after reranking) are used to build citations and generate a grounded answer. The answer is either generated by the server LLM or returned as `retrievedContext` for browser WebLLM.

## Knowledge Vault

The Knowledge Vault is Aivora's document store. Users upload `.txt` or `.md` files via the Admin panel. Aivora:

1. Chunks documents into ~512-token segments with 64-token overlap
2. Embeds each chunk using the local MiniLM embedder
3. Stores embeddings + metadata in Supabase

The Knowledge Vault enables fully grounded answers with source citations. Without documents, Aivora falls back to general WebLLM answers.

## Citation System

Every answer includes source citations with:
- Document title and file name
- Chunk content excerpt (quoted text)
- Relevance score (0–1)
- Page number (if extracted from metadata)
- Source URL (if provided)

Citations are deduplicated and ranked by relevance. The citation system is implemented in `src/lib/ai/citations/source-schema.ts`.

## Privacy Design

- No user data is sent to external AI APIs (when using Local WebLLM Mode)
- WebLLM model runs entirely in the browser — queries and documents never leave the device
- Supabase stores only the user's own uploaded documents
- Service role key is server-only; never exposed to the browser
- No telemetry or analytics are sent anywhere

## Vector Database Schema

**`documents` table:**
- `id` (uuid, primary key)
- `title` (text)
- `file_name` (text)
- `file_type` (text)
- `source_url` (text, nullable)
- `created_at` (timestamptz)

**`document_chunks` table:**
- `id` (uuid, primary key)
- `document_id` (uuid, foreign key → documents)
- `content` (text)
- `embedding` (vector(384))
- `chunk_index` (integer)
- `token_count` (integer)
- `metadata` (jsonb)
- `created_at` (timestamptz)

## Frequently Asked Questions

**Does Aivora require an OpenAI API key?**
No. Aivora is designed to work without any external AI API. It uses browser-local WebLLM (Phi-3.5-mini) for generation and server-local @xenova/transformers for embeddings.

**Does Aivora require Ollama?**
No. Aivora does not depend on Ollama, LM Studio, or any locally running inference server.

**What is the minimum setup to run Aivora?**
You need Node.js, a Supabase project with the provided SQL migrations applied, and the three Supabase environment variables set in `.env.local`.

**Can Aivora work without internet?**
Yes, after the WebLLM model is downloaded on first use, Aivora can answer questions from the Knowledge Vault entirely offline (browser + local Supabase or self-hosted Supabase).

**How does Aivora handle large documents?**
Documents are chunked into ~512-token segments with 64-token overlap before embedding. Only the top-5 most relevant chunks are used for answer generation.

**What file formats does Aivora support?**
Currently `.txt` and `.md` files. PDF support is planned.

**What WebGPU model does Aivora use?**
Phi-3.5-mini-instruct-q4f16_1-MLC, loaded via @mlc-ai/web-llm. The model is ~2GB and is cached in the browser after first download.

**Is Aivora open source?**
Yes. The source code is available at https://github.com/fokrulislam/aivora under an open-source license.
